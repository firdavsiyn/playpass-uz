# ТЗ: Переход модели подписки «часы → визиты»

**Дата:** 10 июня 2026
**Контекст:** BM v1.2 — лимит подписки считается в ВИЗИТАХ, не часах. Сопровождает [BM v1.2 economics](PlayPass_UZ_BM_v1.2_economics.md).

---

## 0. Ключевой факт (упрощает всё)

Чекин **уже** работает по-визитно: `checkin/index.ts` вставляет визит с `hours_spent: 1` и списывает `hours_balance − 1` за каждый заход, независимо от реального времени в клубе. Значит:

- `hours_balance` **физически уже считает визиты**, а не часы.
- Баг — только в **ярлыках UI** («часов» вместо «визитов») и в **калькуляторе экономии** (умножает на часовую ставку, хотя визит — это сессия, а не час).

**Вывод:** не нужно переписывать логику списания. Нужно: (1) переименовать ярлыки, (2) починить калькулятор экономии, (3) обновить значения планов, (4) ограничить Day по времени, (5) убрать безлимит из продажи.

---

## 1. Решение по схеме БД

**НЕ переименовываем колонку `hours_balance` → `visits_balance` в Phase 1.**
Причина: имя колонки тянется через edge-функции, RPC `bump_subscription_hours`, `Subscription.fromJson`, RLS-триггеры. Рейн повышает риск без продуктовой выгоды. Колонка остаётся `hours_balance`, но **семантически = «визиты»**.

- ✅ Phase 1: меняем смысл + ярлыки, колонку не трогаем.
- 🔵 Phase 2 (опционально, потом): чистый рейн `hours_balance → visits_balance` + `hours_spent → не нужен` для ясности кода. Отдельная задача, не блокирует запуск.

## 2. Изменения по слоям

### 2.1 Plans config — `core/constants/app_constants.dart`
Поле `PlanConfig.hours` теперь трактуется как **визиты/мес**. Обновить значения под BM v1.2:

| Код | Было (hours) | Стало (visits) | Цена | allowedSlots |
|---|---:|---:|---:|---|
| `daily` | 4 | 4 (визита, 1 день) | 25 000 | day, evening, night |
| `standard` → переименовать роль в **Day** | 30 | **12** | 149 000 | **только `day`** |
| `pro`/`vip` (unlimited) | 0 | — | — | **убрать из продажи** |
| новый **Anytime** | — | 12 | 249 000 | day, evening, night |

> Решение: либо переиспользовать существующие коды (`standard`→Day, добавить `anytime`), либо ввести чистые коды `day`/`anytime`. Рекомендую чистые коды `day` и `anytime` + оставить старые в БД-констрейнте для legacy-строк.

Безлимит (`isUnlimited: true`) на старте остаётся только у будущих годовых — из активной продажи Pro/VIP-месячный убрать.

### 2.2 Калькулятор экономии — `core/utils/savings_calculator.dart` ⚠️ СЕМАНТИКА
Сейчас: `экономия = визиты × часовая_ставка` — **неверно** (визит ≠ 1 час).
Стало: `экономия = визиты × средняя_стоимость_сессии_кэшем`.

- Ввести `avgСashSessionByPlan` (например ~30 000 сум — средний чек кэш-сессии).
- `formatAmount` оставить.
- На главном «Вы сэкономили» = `кол-во визитов × 30 000 − цена подписки` (или просто `визиты × 30000` как «эквивалент кэшем»).

### 2.3 Edge functions
- **`checkin/index.ts`** — логика списания уже верна (−1 за визит). Убрать только смысловую путаницу: переименовать в ответе `hours_remaining` → `visits_remaining` (и в Dart-парсере). Лимит `maxDailyCheckins`/«1 визит в день» оставить.
- **`payout-calc/index.ts`** — ставка за визит зависит от времени визита: off-peak (08–18) = 10 000, peak = 18 000. Это новое: сейчас фикс `RATE_PER_VISIT_UZS = 8000`. Заменить на time-aware ставку.
- **`rahmat-webhook` / `click-webhook` / `activate_subscription_from_payment`** — при активации ставить `hours_balance = <visits>` из плана (12 для Day/Anytime, 4 для daily, NULL только для будущих годовых-безлимит). Безлимит-ветку (`-1`) на старте не используем для месячных.
- **`user-qr-generate`** — возвращает `hours_balance` в payload; переименовать ярлык в UI, значение оставить.

### 2.4 Dart model — `models/subscription.dart`
- `hoursBalance` / `hoursTotal` — оставить имена полей (мапятся на БД), но геттеры отображения переименовать:
  - `hoursDisplay` → `visitsDisplay` («∞» для будущих безлимит, иначе число визитов).
  - `isUnlimited` — оставить (для годовых).
- `Subscription.fromJson` — без изменений (колонка та же).

### 2.5 UI-строки — `core/l10n/strings_ru.dart` + `strings_uz.dart`
Заменить «часов / ч / часа» → «визитов / визит / визита» во всех ключах подписки:
- `home.*`, `sub.*`, `pay.hours_desc`, `scan.hours_left`, `booking.hours_short` и т.д.
- **Русская плюрализация обязательна:** 1 визит, 2 визита, 5 визитов. Ввести хелпер:
  ```dart
  String pluralVisits(int n) {
    final m10 = n % 10, m100 = n % 100;
    if (m10 == 1 && m100 != 11) return '$n визит';
    if (m10 >= 2 && m10 <= 4 && (m100 < 12 || m100 > 14)) return '$n визита';
    return '$n визитов';
  }
  ```
- Узбекский: плюрализация не нужна («$n tashrif»).

### 2.6 UI-экраны (ярлыки, значения остаются)
Точки из аудита — заменить отображение «ч/часов» на визиты:
`home_screen.dart`, `subscription_widget.dart`, `recent_visits_widget.dart`, `visit_history_screen.dart`, `savings_screen.dart`, `qr_scanner_screen.dart` (после чекина: «осталось N визитов»), `friends_screen.dart`, `gift_purchase_screen.dart`, `story_generator.dart` (картинка для шеринга).
Фичефлаг-выключенные (`booking_screen`, `loyalty_screen`) — низкий приоритет.

### 2.7 SQL
- `subscriptions.hours_balance` — комментарий поправить на `-- now counts VISITS`.
- Месячная функция экономии (`initial_schema.sql:188`, `SUM(hours_spent)`) — если используется, трактует визиты; для денежной экономии умножать на стоимость сессии, не на час.
- `bump_subscription_hours` RPC — имя оставить (реферал +начисляет визиты), смысл = визиты.

## 3. Время-ограничение Day (off-peak)
Серверная проверка в `checkin/index.ts`: если план = `day` и текущий час вне 08–18 → отказ «Дневной тариф действует с 08:00 до 18:00». Не полагаться только на клиентский `allowedSlots`.
