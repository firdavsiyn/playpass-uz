## 1. Суть

PlayPass — «1FIT для геймеров». Одна подписка = доступ в сеть компьютерных клубов Ташкента по QR, без привязки к одному заведению. Клубу — только распечатанный постер, веб-админка показывает визиты.

**Два потока выручки:**
- **Месячные подписки** — для регулярных, живут в off-peak (дневные «мёртвые часы» клубов).
- **Разовые пассы** — для пика/ночи/выходных, высокая маржа, ноль риска по марже.

## 4. Модель выплаты клубу (фикс за визит)

Привязана к реальной кэш-стоимости сессии. Off-peak = found money (пустое кресло, клуб платит за электричество в любом случае → берёт долю). Пик/VIP = вытесняет платящего клиента → выплата близка к кэшу.

| Сценарий | Кэш-стоимость (2-3ч) | **Фикс-выплата/визит** | % от кэша |
|---|---|---:|---:|
| Off-peak Main, день | 32–48к | **9 000** | ~20–28% |
| Пик / вечер (Main/Std) | 40–75к | **14 000** | ~25–30% |
| VIP / ночь | 60–105к | **22 000–30 000** | ~30–40% |

⚠️ Зафиксировать письменно с каждым пилотным клубом. Модель терпит off-peak до ~10 500, но не на тарифе Standard.

---

## 5. Золотое правило платёжеспособности

> **выплата_за_визит × лимит_визитов < цена_подписки**

Если держится — PlayPass в плюсе даже при полном выжимании капа. Лимит визитов = страховка маржи. Прибыль приходит от breakage (никто не выжимает кап полностью).

**Безлимит-частота забанена.** Доказательство: «Daily VIP» 300к / 30 виз / 25к выплата → худший случай **750к** при цене 300к = **−150%**. Математически невозможен под фикс.

- CAC: реферал +визиты + Story-Generator, K-factor цель 0,5 → effective CAC ниже на ~⅓. Ориентир 30–50к.
- **LTV/CAC ≈ 7–10×.**

## 10. GTM (cold-start через off-peak клин)

1. **Один район** Ташкента (центр / плотный спальный).
2. **5–10 клубов** с самыми мёртвыми дневными часами → подписать на фикс за off-peak визит (9к). Договор партнёра готов.
3. Старт — **Lite + Plus + 3 разовых пасса**. Только дневное окно для Lite.
4. Сегмент спроса: студенты / фрилансеры / дневная молодёжь.
5. Расширять (Student-тариф, вечер, VIP-зоны, районы) — после того, как flywheel в первом районе закрутился.

## 12. Допущения к валидации (флаги)
1. **58% использования** — главный рычаг. При 100% Standard → +14,8%. Мерить redemption с месяца 1.
2. **VIP-цены (25–35к) и пакеты** — только данные основателя, веб не подтвердил. Проверить с 2–3 клубами до печати цены ночного пасса.
3. **Выплата 9к off-peak** — зафиксировать письменно. Модель терпит до ~10,5к (кроме Standard).
4. **40% attach-rate разовых** в Y1-прогнозе — иллюстративно, данных нет.
5. **Дневной спрос** в выбранном районе — экзистенциальный риск cold-start. Проверяется пилотом, не моделью.
